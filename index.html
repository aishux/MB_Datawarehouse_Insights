Key Country Observations

🏆 UAE is the #1 market by every measure: 126,558 accounts (13% of total), 25% funded rate (above average), $36.4M NMI (52% of all 2026 NMI), and the highest GA traffic at 397k sessions.
🇨🇦 Canada punches above its weight: 31.8% funded rate — the highest of any major market — and $8.3M NMI. High-value, high-intent audience.
🇮🇹 Italy has exceptional conversion: 33.5% funded rate (best of large markets) despite only $395 average account size. Volume conversion machine.
⚠️ USA is a major red flag: 17,775 accounts but only 426 funded = 2.4% conversion. The worst funded rate by far. Possible regulatory restrictions, wrong audience targeting, or product-market fit issues.
⚠️ Negative NMI in several markets: France (−$326K), UK (−$1.16M), Mexico (−$1.13M), Italy (−$2.92M), Pakistan (−$547K), Vietnam (−$84K). These markets are cash-negative in 2026 YTD — may be new or promotional.
🇲🇽 Mexico bounce rate 86.6%: Abnormally high. Almost 9 in 10 sessions bounce. This suggests a serious landing page relevance or localisation problem for Mexican traffic.

Data Quality Issues Found

These are real problems identified in the raw data. They affect how much you should trust certain KPIs and rows. None of these are Claude's limitations — they are properties of the source data.
#FaultCategoryExamples from DataWhy It MattersKPIs Affected
1 Clicks Exceeding Impressions Funnel Logic Denmark: 1 impression, 29 clicks. Netherlands: 17 impressions, 266 clicks. Physically impossible in any ad system. CTR becomes meaningless (over 100%). Suggests double-counting of clicks, pixel misconfigurations, or platform reporting bugs. CTR, CPC
2 Clicks with Zero Impressions Funnel Logic Afghanistan: 0 impressions, 250 clicks. Albania: 0 impressions, 14 clicks. Clicks cannot physically occur without an impression first. Distorts CTR to undefined/infinite. Indicates tracking or reporting integration errors between ad platform and warehouse ingestion. CTR, LPV %
3 Deposits Column Inflated 100x in accounts_python Data Governance accounts_python.total_deposits_usd sums to $237 billion. dm_profile.total_deposit_usd (deduplicated per user) sums to $2.32 billion — a 100x difference. Using the wrong column would overstate average account size by 100x and completely invalidate ROI calculations. Always use dm_profile for deposits. Avg Account Size, ROI
4 Inconsistent Country Name Formats Across Tables Reporting Same country in 3 formats: "Cote DIvoire", "Cote d'Ivoire", "Côte d'Ivoire". accounts_python uses ISO2 codes, accounts_daily uses full names, paidmarketing uses regional labels (Worldwide, MENA, KSA). Prevents country-level cross-source joins. Marketing cost cannot be attributed to individual countries. Any by-country report mixing these tables will double/triple count or drop rows. CPA, CPFA, ROI (by-country), LPV % (by-country)
5 Missing Country on Rows with Performance Data Data Governance Row 67 in the QA source data: Country is completely blank but has Leads = 11. Revenue/lead data cannot be attributed geographically. Creates "unknown" buckets that distort regional totals. By-country breakdowns
6 Sessions Exceeding Clicks in Some Countries Attribution Ireland: 47 clicks but 974 GA sessions. Sessions from paid ads should be ≤ clicks unless organic/direct traffic is mixed into the GA view without proper segmentation. LPV % calculated as GA views ÷ paid clicks becomes meaningless for these rows. Indicates attribution bleed between paid and organic/direct channels in GA. LPV %
7 Inconsistent Percentage Formatting Reporting Some percentage columns contain decimals (0.104 = 10.4%), others contain already-multiplied values (CTR % = 29 meaning 29%). LPV % values range from 0.45 to 11.16 with no clear unit. Users reading raw data could misinterpret values by a factor of 100. For example, a 0.2 in "Leads to Accounts %" actually means 20%, not 0.2%. CTR, LPV %, Bounce Rate (raw data layer)
8 GA Data Loaded via Manual CSV (Fragile Pipeline) Data Governance The stg_ga_csv table has phantom Unnamed: columns — a signature of pandas CSV import. This means someone is manually exporting from GA and uploading the file. No automated pipeline exists. GA data could be stale, missing recent weeks, or accidentally loaded twice. LPV % and Bounce Rate metrics depend entirely on this fragile process. LPV %, Bounce Rate
9 Georgia CTR Anomaly (3.22% / $0.04 CPC) Funnel Logic Georgia targeting geo: 5.1M clicks from 158.9M impressions at $216k cost. CTR = 3.22%, CPC = $0.04. For financial services, this is 3–5x the normal CTR at 1/20th the normal CPC. Strongly suggests click fraud, bot traffic, or a pipeline tagging error. If real, it skews total aggregate CTR downward and CPC upward. Needs investigation and potential quarantine. CTR, CPC (aggregate distortion)

SQL Queries — Every KPI

Each query below is exactly what produced the verified 2026 numbers, filtered to 2026-01-01 → 2026-06-25 using each table's own date column.
Q1 — NMI $71.58M
SELECT SUM(daily_nmi_usd) AS nmi_2026
FROM dm_sales_performance_2026
WHERE reporting_date >= '2026-01-01' AND reporting_date < '2026-06-26';
Q2 — Accounts opened, funded & Account→Funded % 73,407 · 21,825 · 29.73%
SELECT
  SUM(is_test=0)                                                                       AS accounts_opened,
  SUM(is_test=0 AND Account_Type IN ('Individual Client','Introducing Broker'))        AS client_ib_accts,
  SUM(is_test=0 AND has_ftd=1)                                                         AS funded_accounts,
  SUM(is_test=0 AND has_ftd=1) / NULLIF(SUM(is_test=0),0)                              AS account_to_funded
FROM accounts_python
WHERE account_creation_time >= '2026-01-01' AND account_creation_time < '2026-06-26';
Q3 — Marketing spend (paid + mobile + offline) $1.56M
SELECT
  (SELECT SUM(Cost)  FROM paidmarketing_2024          WHERE Date >= '2026-01-01' AND Date < '2026-06-26') AS paid_cost,    -- 1,124,997
  (SELECT SUM(Cost)  FROM adjust                      WHERE Date >= '2026-01-01' AND Date < '2026-06-26') AS adjust_cost,  --    46,602
  (SELECT SUM(spend) FROM stg_offline_marketing_spend WHERE date >= '2026-01-01' AND date < '2026-06-26') AS offline_cost; --   391,338
-- total marketing spend 2026 = 1,562,937
Q4 — CPA & CPFA $21.43 · $71.61
-- Uses total spend from Q3 (1,562,937) over 2026 account counts from Q2
SELECT
  1562937 / NULLIF(SUM(is_test=0 AND Account_Type IN ('Individual Client','Introducing Broker')),0) AS cpa,
  1562937 / NULLIF(SUM(is_test=0 AND has_ftd=1),0)                                                  AS cpfa
FROM accounts_python
WHERE account_creation_time >= '2026-01-01' AND account_creation_time < '2026-06-26';
Q5 — Deposits, funded users & Avg Account Size $114.06M · 17,329 · $6,582
SELECT
  COUNT(*)                                    AS funded_users,
  SUM(total_deposit_usd)                      AS deposits,
  SUM(total_deposit_usd) / NULLIF(COUNT(*),0) AS avg_account_size
FROM dm_profile
WHERE ftd_dtm >= '2026-01-01' AND ftd_dtm < '2026-06-26';
-- user grain (dm_profile) — avoids the ~100x multi-account inflation in accounts_python
Q6 — CTR & CPC 0.569% · $1.50
SELECT
  SUM(Clicks)                            AS clicks,        --     749,057
  SUM(Impressions)                       AS impressions,   -- 131,706,139
  SUM(Cost)                              AS paid_cost,     --   1,124,997
  SUM(Clicks)/NULLIF(SUM(Impressions),0) AS ctr,           --      0.569%
  SUM(Cost)/NULLIF(SUM(Clicks),0)        AS cpc            --       $1.50
FROM paidmarketing_2024
WHERE Date >= '2026-01-01' AND Date < '2026-06-26';
Q7 — Leads 67,792 · 15,577 funded
SELECT
  COUNT(*)         AS leads,
  SUM(has_ftd=1)   AS leads_funded
FROM leads_python
WHERE creation_time >= '2026-01-01' AND creation_time < '2026-06-26';
-- demo flag (is_demo) is not populated, so this counts all 2026 registrations
